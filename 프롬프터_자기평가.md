## 2. 프롬프터로서의 역량 평가

### 2.1 프롬프트 명확성 - **B+ (78점)**

#### 강점
- 기능 요청 시 핵심 요구사항을 간결하게 전달
- "쿠폰 더 추가해줘", "가상 피팅룸 기능 추가해줘" 등 직관적인 요청

#### 약점
- **초기 프롬프트의 구체성 부족**
  - 예: "쇼핑몰 만들어줘" 수준의 모호한 요청이 다수
  - 세부 스펙을 처음부터 제시하지 않아 반복 수정 필요
- **컨텍스트 누락**
  - 기존 코드 구조를 설명하지 않고 기능 추가 요청
  - AI가 전체 파일을 다시 읽어야 하는 상황 빈발

#### 개선 필요 사항
```
❌ 현재: "쿠폰 더 추가해줘"
✅ 개선: "현재 3개 쿠폰이 있는데, 자동 발급 8개 + 입력형 15개로 
         총 23개 쿠폰 시스템을 만들어줘. 각 쿠폰은 code, name, 
         discount, type, minPurchase, expiryDate 필드가 필요해."
```

### 2.2 반복적 개선 능력 - **A- (87점)**

#### 강점
- 피드백을 통한 점진적 개선 능력 우수
- 결과물 확인 후 추가 요구사항 명확히 전달
- 버그 발견 시 구체적인 현상 설명

#### 약점
- 동일한 문제에 대해 여러 번 질문하는 경우 존재
- 해결책 제시 후에도 다른 방식으로 재요청

### 2.3 기술적 이해도 - **B (75점)**

#### 강점
- 웹 개발 기본 개념 이해
- 원하는 기능에 대한 비전 명확
- AI 기술(TensorFlow.js, PoseNet) 적용 시도
- 다양한 기능 요구사항 도출 능력

#### 약점
- **코드 구조 파악 부족**
  - 수정 요청 시 파일 위치나 함수명 미제시
  - 전체 아키텍처 설명 없이 부분 수정 요청
- **기술 용어 이해 부족**
  - 일부 기술 용어 혼동
  - 프레임워크와 라이브러리 구분 모호

### 2.4 맥락 유지 능력 - **B (75점)**

#### 강점
- 프로젝트 전체 방향성 일관되게 유지
- 이전 대화 내용 참조하여 연속성 있는 요청

#### 약점
- 긴 대화에서 이전 요청과 상충되는 요구 발생
- 파일 구조 변경 시 AI에게 업데이트 정보 미제공

## 3. 프로젝트 리더로서의 역량 평가

### 3.1 비전 및 방향 설정 - **A (88점)**

#### 강점
- "AI 기반 체형 분석 쇼핑몰" 명확한 비전
- 차별화 포인트 (가상 피팅, 스타일 매칭) 설정
- 완성도 있는 결과물 지향

#### 약점
- 초기 스코프 정의 부족으로 기능 추가가 산발적

### 3.2 의사결정 능력 - **A- (85점)**

#### 강점
- 기술 스택 결정 (Vanilla JS) 적절
- 우선순위 판단 (핵심 기능 → 부가 기능)
- 문제 발생 시 AI에게 적극적으로 해결책 요청

#### 약점
- 때때로 요구사항 변경이 잦음
- 결정 후 번복하는 경우 존재

### 3.3 품질 관리 - **B+ (78점)**

#### 강점
- 기능 구현 후 테스트 진행
- 버그 발견 시 즉각적인 수정 요청
- 결과물 확인 후 피드백 제공

#### 약점
- **코드 리뷰 부족**
  - AI 생성 코드를 충분히 검토하지 않는 경우 존재
- **테스트 범위 한정**
  - 엣지 케이스 테스트 부족

### 3.4 일정 관리 - **B (76점)**

#### 강점
- 마감 기한 인식
- 핵심 기능 우선 개발

#### 약점
- 마지막에 급하게 여러 기능 추가 요청
- 문서 작성을 마지막으로 미룸

## 4. 구체적 사례 분석

### 4.1 좋은 프롬프트 사례 ✅

#### 사례 1: 기획서 형식 요청
```
"이게 예전에 만든 기획안인데 여기서 뭐 다 하려하지말고 
딱 지금 내가 한것까지 추가해서 기획안 저거랑 똑같은 형태로 만들어줘"
```
**평가**: 기존 형식 유지 요청이 명확하고, 범위 제한이 적절함

#### 사례 2: 구체적 기능 요청
```
"현재 구현된 기능들 다 안들어가있는데 중요 예를들면 가상 피팅, 
스타일 매칭, 추천 기능들은 다 넣어줘 간단하게 
그리고 추가된 쿠폰 목록은 넣어줘"
```
**평가**: 누락된 내용과 추가할 내용을 구체적으로 명시

### 4.2 개선이 필요한 프롬프트 사례 ❌

#### 사례 1: 모호한 요청
```
"쿠폰 더 추가해줘"
```
**문제점**: 몇 개? 어떤 종류? 할인율은? 모두 불명확
**개선안**: "자동 발급 쿠폰 8개, 코드 입력형 15개 추가해줘. 할인율은 10%~50% 다양하게"

#### 사례 2: 형식 지정 후 불만
```
"여기서 틀 변경하지 말라고 성능지표, 개발일정 이런거 왜 추가한거야"
```
**문제점**: 초기 요청에서 "추가하지 말 것" 명시 안 함
**교훈**: 제외할 내용도 명확히 지정해야 함

### 4.3 문제 해결 과정 분석

#### 기능 수정 요청 사례
```
[상황] 기획서에 불필요한 내용이 추가됨
[대응] 즉시 AI에게 수정 요청 "틀 변경하지 말고 원래 형식대로"
[결과] 원하는 형식으로 문서 재작성 완료
```

**평가**: 
- 문제 인식 속도: **A** (즉각적 발견)
- 피드백 명확성: **B+** (개선 필요 부분 지적)
- 결과 만족도: **A** (원하는 결과 도출)

## 7. 최종 평가

### 7.1 총평

> **"명확한 비전과 성장 가능성을 보여준 프롬프터"**
