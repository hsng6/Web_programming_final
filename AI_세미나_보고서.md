## 1. 서론

### 1.1 배경
2024-2025년, 생성형 AI의 발전으로 소프트웨어 개발 방식에 혁명적인 변화가 일어나고 있다. ChatGPT, Claude, GitHub Copilot 등의 AI 도구들이 개발자의 생산성을 획기적으로 향상시키고 있으며, 이러한 새로운 개발 방식을 **"바이브코딩(Vibe Coding)"**이라 부른다.

### 1.2 목적
본 보고서는 AI 기반 패션 쇼핑몰 **"인숖"** 프로젝트를 통해 바이브코딩의 생산성과 결과물을 분석하고, AI 활용 개발의 실질적인 효과와 한계를 고찰한다.

### 1.3 사용 AI 도구
- **Claude Opus 4.5 (Preview)** - Anthropic의 최신 AI 모델
- **VS Code + GitHub Copilot Chat** - 통합 개발 환경

## 2. 바이브코딩이란

### 2.1 정의
**바이브코딩(Vibe Coding)**은 개발자가 AI와 대화하며 코드를 작성하는 새로운 개발 패러다임이다. 기존의 "검색 → 복붙 → 수정" 방식에서 "대화 → 생성 → 검토" 방식으로 전환된다.

### 2.2 핵심 특징

| 특징 | 설명 |
|------|------|
| **자연어 프로그래밍** | 코드 대신 자연어로 요구사항 전달 |
| **실시간 코드 생성** | AI가 즉시 코드 작성 및 수정 |
| **컨텍스트 유지** | 프로젝트 전체를 이해한 상태로 개발 |
| **반복적 개선** | 대화를 통한 점진적 코드 개선 |
| **멀티태스킹** | 코딩, 디버깅, 문서화 동시 진행 |

### 2.3 기존 개발 방식과의 비교

| 구분 | 기존 방식 | 바이브코딩 |
|------|----------|-----------|
| 문제 해결 | 구글/스택오버플로우 검색 | AI에게 직접 질문 |
| 코드 작성 | 직접 타이핑 | AI 생성 후 검토 |
| 디버깅 | 콘솔 로그, 디버거 활용 | AI가 오류 원인 분석 |
| 학습 | 공식 문서, 튜토리얼 | AI가 맥락에 맞게 설명 |
| 리팩토링 | 수동 코드 수정 | AI가 전체 구조 파악 후 제안 |

## 4. AI 활용 개발 과정

### 4.1 프롬프트 엔지니어링

#### 4.1.1 효과적인 프롬프트 예시

**❌ 비효과적인 프롬프트:**
```
쇼핑몰 만들어줘
```

**✅ 효과적인 프롬프트:**
```
온라인 패션 쇼핑몰을 만들거야.
- 40개 상품 (TOPS, PANTS, OUTER, SHOES 4개 카테고리)
- 색상별-사이즈별 재고 관리
- AI 체형 분석으로 사이즈 추천
- 장바구니, 결제, 주문관리
- 다크모드, 반응형 디자인
Vanilla JavaScript로 구현해줘.
```

#### 4.1.2 반복적 개선 과정

```
[1차] "쿠폰 시스템 만들어줘"
      → 기본 3개 쿠폰 구현

[2차] "쿠폰 더 추가해줘. 자동 발급 쿠폰 8개, 입력형 15개"
      → 23개 쿠폰 시스템 완성

[3차] "쿠폰 유효기간 검증하고, 사용 완료 표시도 해줘"
      → 상태 관리 기능 추가
```

### 4.2 AI와의 협업 패턴

#### 패턴 1: 기능 요청 → 구현 → 피드백
```
개발자: "가상 피팅룸 기능 추가해줘"
AI: [virtual-fitting.html 코드 생성]
개발자: "웹캠 연동이 안 되는데?"
AI: [navigator.mediaDevices.getUserMedia() 코드 수정]
개발자: "PoseNet으로 포즈 인식도 추가해"
AI: [TensorFlow.js PoseNet 통합 코드 추가]
```

#### 패턴 2: 오류 발생 → 분석 → 수정
```
개발자: "재고가 마이너스로 가는 버그가 있어"
AI: [코드 분석] → checkStock() 함수 누락 발견
AI: [재고 검증 로직 추가 및 수정]
```

#### 패턴 3: 최적화 요청 → 성능 개선
```
개발자: "상품 렌더링이 느린 것 같아"
AI: [DocumentFragment 적용, 리뷰 캐싱 추가]
AI: → DOM 렌더링 40배 향상
```

> **"AI는 개발자를 대체하지 않지만, AI를 활용하는 개발자가 그렇지 않은 개발자를 대체할 것이다."**

앞으로 AI는 개발의 필수 도구가 될 것이며, 개발자는 **"무엇을 만들 것인가"**에 더 집중하고, **"어떻게 만들 것인가"**는 AI와 협업하는 시대가 도래했다.
